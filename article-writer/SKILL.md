# Article Writer

## Overview

专业文章写作助手，支持多平台内容创作。根据用户需求完成从受众分析、大纲生成、文章撰写到润色完善的完整流程。

## Supported Platforms

### 小红书
- 标题党化、吸引眼球
- 表情符号丰富
- 段落短小精悍
- 实用干货+个人体验结合
- 字数：500-1000字

### 推特 (X)
- 280字符限制
- 简洁有力
- 话题标签
-  Threads 形式
- 字数：单条50-280字，系列可多篇

### 微信公众号
- 深度内容
- 正式但不死板
- 标题吸引人
- 图文并茂
- 字数：1000-3000字

## 工作流程

### Step 1: 受众分析

向用户询问以下问题：

**基础信息**
- 目标受众是谁？（年龄、职业、兴趣）
- 读者对你 topic 的了解程度？
- 读者最关心什么痛点？

**写作目的**
- 你想传达什么核心信息？
- 希望读者看完后做什么？

**风格偏好**
- 正式还是轻松？
- 幽默还是严肃？
- 口语化还是书面化？

### Step 2: 大纲生成

根据受众分析和主题，生成结构化大纲：

**大纲要素**
- 标题（吸引点击）
- 引言（引起兴趣）
- 主体（3-5个核心点）
- 结尾（行动号召或总结）

**大纲格式**
```
# [文章标题]
## I. 引言
- 痛点引入
- 文章承诺

## II. 核心论点1
- 子观点1.1
- 子观点1.2

## III. 核心论点2
...

## IV. 总结
- 核心要点回顾
- 行动号召
```

**输出格式**
用 Markdown 格式输出大纲，让用户确认或修改。

### Step 3: 文章撰写

用户确认大纲后，应用以下写作技巧：

#### 句子技巧

**句子长度变化**
- 长短句交替，创造音乐感
- 短句有力，长句积累势能
- 大声朗读，感受节奏

**示例**
- （原）"他拿起书开始读。他放不下了。他完全入迷了。几小时不知不觉过去了。"
- （改）"一拿起书，他就发现自己再也放不下了。他彻底沉浸其中，几小时不知不觉就溜走了。"

#### 遣词造句

**主动语态优先**
- 把读者的注意力引向"行动者"
- "消防员救下猫" > "猫被消防员救下"

**避免重复**
- 不要连续以同一词开头
- 避免同一结构的重复

**示例**
- （原）"她打开门。她看到他站在那里。她感觉心跳加速。她不敢相信他真的在这里。"
- （改）"一打开门，她就看到他站在那里。她心跳加速，简直不敢相信他真的来了。"

#### 内容具体化

**走向抽象金字塔底层**
- "一只贵宾犬" > "一只狗"
- "他双手颤抖、额头出汗" > "他很紧张"

**展示而非讲述**
- 添加感官细节
- 用行动和对话替代情感描述

**示例**
- （原 - 讲述）"他在演讲前很紧张。"
- （改 - 展示）"他准备演讲时，双手颤抖，额头上渗出了汗珠。"

#### 平台适配

**小红书特技**
- 标题：数字+痛点+解决方案
- 表情：💡🔥📌✨✅🎯
- 开头：痛点引入+个人经历
- 结尾：互动引导（"你们觉得呢？"）

**推特特技**
- 开头：钩子+核心观点
- 中间：1-2个有力论据
- 结尾：话题标签

**微信公众号特技**
- 标题：悬念/数字/对比
- 开头：故事/问题/数据引入
- 主体：论点+案例+数据
- 结尾：总结+互动+推荐阅读

### Step 4: Humanizer 润色

使用 Humanizer 技能去除 AI 写作痕迹，让文章更自然：

**Humanizer 命令**
```
/humanizer-zh
请帮我润色以下文章，让它更自然、更有"人味"：
[粘贴文章内容]
```

**Humanizer 能修复的 AI 痕迹**

📝 **内容模式**
- 过度强调意义、遗产和更广泛的趋势
- 提纲式的"挑战与未来展望"部分
- 宣传和广告式语言

🔤 **语言语法模式**
- 过度使用的"AI 词汇"（此外、至关重要等）
- 系动词回避
- 三段式法则过度使用

🎨 **风格模式**
- 破折号过度使用
- 粗体过度使用
- 内联标题垂直列表

💬 **交流模式**
- 填充短语
- 谄媚/卑躬屈膝的语气
- 通用积极结论

在输出终版前，检查以下要点：

- [ ] 句子长度有变化
- [ ] 主动语态为主
- [ ] 内容具体化
- [ ] 无连续重复开头
- [ ] 符合平台特点

- [ ] 错别字检查
- [ ] 标点符号规范
